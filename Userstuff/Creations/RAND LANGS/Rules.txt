Samster is a language derived from Assembly, it has NO currently working compiler/interpreter.
Samster looks like Assembly but i'd say its easier, atleats in loops. As of now Samster has NO
loops. Every statement must end with a ';'.

VALUES:
#i32 - Integer
'#'f64 - Float, the '' is not neccesary.
#String - strings.
#Char - 1 character.
#lst - Lists. 
#Rtn

COMMANDS:
mov - moves something into a registor.

--add, min, mult, div - adds, subtracts, multiplies, divides 2 registor.
-SYNTAX:
[operation] [first reg], [sec reg], [third and so on or result reg]; #Int
Last register is the input, others are the adding registors.

-addS - used with @conc to move a string into a int type reg.
 
-prt - prints a value, needs perm: I/O for access.

--cmp - compares value.
-Compares two value, as said below use one of the return type reg to hold the returns.
returns are: -1: less than, 1: greater than, 0: equal to. This is only applicable in cmp.
SYNTAX:
cmp [reg 1], [reg 2] -> [return reg]; #[type]
Example:
mov 21: R1; #i32
mov -23: R2; #i32
cmp R1, R2 -> T1; #Rtn

-chk - checks a return type registor;

CONTROL FLOW:
These are the main things:
-jump_if_0, jump_if_1, jump_if_-1, jump_if_cancelled.
to access the return you use: __eq, __gt, __lt.
how to use them:
SYNTAX:
mov 1: R1; #i32
mov 2: R2; #i32
cmp R1, R2 -> T1; #Rtn                 (verbal expression: compare R1 WITH R2, is it bigger/small/equal?)
chk T1:

    [jump cmd] -> [dest, one of the "return access"],
    [jump cmd] -> [dest, one of the "return access"],
__[dest](T1){
    [code]
};
__[dest](T1){

};
__[dest](T1){
    [code]
};


MAIN CODE:
__start: - starts a code execution.
__stop; - stops the code execution.
__ask: [name of what you want to ask]{} - to ask the OS/System for permissions etc.


'@' COMMANDS:
@Index - adds indexes to a registor with type list.
@conc - adds a string value with a Int/Float and returns a string.
@crt -used to create new reg,
USAGE:
@crt reg: [name][num1..num2]; #[type]
example:
@crt reg: Y, 1..6; #Int
@crt reg: Z, 1..4; #rtn

REGISTORS:
-there can be 5 types of each registor:
example: (name of registor)1, (name of registor)2, (name of registor)3, (name of registor)4...

-As of now, there can be 6 of 4 registors: R1-6, E1-6, T1-6, S1-6.

-R, E is for normal use like for holding value etc.
-T, S are for holding returns like 0, -1, 1.

-normal value to be entered into registors are: Int.

-After getting a value into a registor we must tell the type of value by:
_-mov [value], [registor]; #[type]

-To get a registor have a different value, use:
_-mov $__[type of value]__:[value]: [registor]; #[type]
